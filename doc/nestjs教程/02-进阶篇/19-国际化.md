# 国际化

国际化（Internationalization，简称i18n）是指设计和开发能够适应不同语言和文化习惯的应用程序的过程。NestJS提供了强大的国际化支持，使开发者能够轻松地为应用添加多语言支持。

## 1. 国际化概述

### 1.1 什么是国际化

国际化是指应用程序能够支持多种语言和地区，而无需修改代码。国际化包括以下几个方面：

- **多语言支持**：应用能够显示不同语言的文本
- **区域设置支持**：应用能够根据不同地区的习惯格式化日期、时间、数字、货币等
- **文本提取**：能够将应用中的文本提取到外部文件，便于翻译
- **动态切换**：能够在运行时动态切换语言

### 1.2 NestJS国际化解决方案

NestJS官方推荐使用 `nestjs-i18n` 库来实现国际化功能。这个库提供了：

- 强大的翻译功能
- 支持多种翻译文件格式（JSON、YAML、PO等）
- 支持参数化翻译
- 支持复数形式
- 支持嵌套翻译
- 支持区域设置检测
- 支持动态切换语言

## 2. 安装与配置

### 2.1 安装依赖

```bash
npm install nestjs-i18n
```

### 2.2 配置翻译文件

1. 在项目根目录创建 `i18n` 目录，用于存放翻译文件：

```
i18n/
├── en/
│   ├── translation.json
│   └── user.json
└── zh/
    ├── translation.json
    └── user.json
```

2. 创建翻译文件内容：

**en/translation.json**：
```json
{
  "welcome": "Welcome to NestJS!",
  "hello": "Hello {name}!",
  "apples": "You have {count, plural, =0 {no apples} =1 {one apple} other {{count} apples}}",
  "nested": {
    "message": "This is a nested message"
  }
}
```

**zh/translation.json**：
```json
{
  "welcome": "欢迎来到NestJS！",
  "hello": "你好 {name}！",
  "apples": "你有 {count, plural, =0 {没有苹果} =1 {一个苹果} other {{count} 个苹果}}",
  "nested": {
    "message": "这是一个嵌套消息"
  }
}
```

### 2.3 配置i18n模块

在 `app.module.ts` 中配置i18n模块：

```typescript
import { Module } from '@nestjs/common';
import { I18nModule } from 'nestjs-i18n';
import * as path from 'path';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [
    I18nModule.forRoot({
      fallbackLanguage: 'en',
      loaderOptions: {
        path: path.join(__dirname, '/i18n/'),
        watch: true,
      },
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

## 3. 使用国际化功能

### 3.1 在控制器中使用

```typescript
import { Controller, Get, Query } from '@nestjs/common';
import { I18n, I18nContext } from 'nestjs-i18n';

@Controller()
export class AppController {
  @Get()
  getHello(@I18n() i18n: I18nContext, @Query('name') name?: string) {
    return {
      welcome: i18n.t('translation.welcome'),
      hello: i18n.t('translation.hello', { args: { name: name || 'World' } }),
      apples: i18n.t('translation.apples', { args: { count: 3 } }),
      nested: i18n.t('translation.nested.message'),
    };
  }
}
```

### 3.2 在服务中使用

```typescript
import { Injectable } from '@nestjs/common';
import { I18nService } from 'nestjs-i18n';

@Injectable()
export class AppService {
  constructor(private readonly i18n: I18nService) {}

  getWelcomeMessage(lang: string): string {
    return this.i18n.translate('translation.welcome', { lang });
  }

  getHelloMessage(name: string, lang: string): string {
    return this.i18n.translate('translation.hello', { args: { name }, lang });
  }
}
```

### 3.3 在管道中使用

```typescript
import { PipeTransform, Injectable, BadRequestException } from '@nestjs/common';
import { I18nService } from 'nestjs-i18n';

@Injectable()
export class ValidationPipe implements PipeTransform {
  constructor(private readonly i18n: I18nService) {}

  transform(value: any) {
    if (!value) {
      throw new BadRequestException(
        this.i18n.translate('validation.required', { lang: 'en' })
      );
    }
    return value;
  }
}
```

## 4. 语言检测

### 4.1 从请求头检测语言

`nestjs-i18n` 会自动从请求头 `Accept-Language` 中检测语言。你可以通过在模块配置中添加 `parserOptions` 来配置：

```typescript
I18nModule.forRoot({
  fallbackLanguage: 'en',
  loaderOptions: {
    path: path.join(__dirname, '/i18n/'),
    watch: true,
  },
  parserOptions: {
    useAcceptLanguageHeader: true,
  },
}),
```

### 4.2 从查询参数检测语言

```typescript
I18nModule.forRoot({
  fallbackLanguage: 'en',
  loaderOptions: {
    path: path.join(__dirname, '/i18n/'),
    watch: true,
  },
  parserOptions: {
    useQueryString: true,
    queryStringParamName: 'lang',
  },
}),
```

这样就可以通过 `?lang=en` 或 `?lang=zh` 来切换语言。

### 4.3 从路由参数检测语言

```typescript
I18nModule.forRoot({
  fallbackLanguage: 'en',
  loaderOptions: {
    path: path.join(__dirname, '/i18n/'),
    watch: true,
  },
  parserOptions: {
    usePath: true,
    pathParamName: 'lang',
  },
}),
```

这样就可以通过 `/en/hello` 或 `/zh/hello` 来切换语言。

## 5. 高级功能

### 5.1 复数形式

如前面的例子所示，`nestjs-i18n` 支持复数形式：

```json
{
  "apples": "You have {count, plural, =0 {no apples} =1 {one apple} other {{count} apples}}"
}
```

### 5.2 嵌套翻译

支持嵌套的翻译键：

```json
{
  "nested": {
    "message": "This is a nested message"
  }
}
```

使用方式：
```typescript
i18n.t('translation.nested.message');
```

### 5.3 动态加载翻译文件

`nestjs-i18n` 支持动态加载翻译文件，这对于大型应用来说非常有用：

```typescript
I18nModule.forRoot({
  fallbackLanguage: 'en',
  loaderOptions: {
    path: path.join(__dirname, '/i18n/'),
    watch: true,
    dynamicLoad: true,
  },
}),
```

### 5.4 自定义翻译加载器

如果默认的文件加载器不满足需求，你可以实现自定义加载器：

```typescript
import { I18nLoader } from 'nestjs-i18n';

@Injectable()
export class CustomLoader implements I18nLoader {
  async load() {
    // 从数据库或其他来源加载翻译
    return {
      en: {
        translation: {
          welcome: 'Welcome to NestJS!',
        },
      },
      zh: {
        translation: {
          welcome: '欢迎来到NestJS！',
        },
      },
    };
  }
}
```

然后在模块配置中使用：

```typescript
I18nModule.forRoot({
  fallbackLanguage: 'en',
  loader: CustomLoader,
}),
```

## 6. 最佳实践

### 6.1 翻译文件组织

- 按功能模块组织翻译文件，如 `user.json`、`product.json` 等
- 使用清晰的命名约定，避免翻译键冲突
- 保持翻译文件的结构一致

### 6.2 翻译键命名

- 使用语义化的翻译键，如 `user.create.success` 而不是 `msg1`
- 使用点分隔符组织翻译键，便于维护
- 避免使用过长的翻译键

### 6.3 性能考虑

- 对于大型应用，考虑使用缓存机制
- 避免在高频调用的地方动态生成翻译
- 考虑使用按需加载翻译文件

### 6.4 测试

- 测试不同语言下的应用行为
- 测试语言切换功能
- 测试翻译缺失情况
