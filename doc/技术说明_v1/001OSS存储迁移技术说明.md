# OSS存储迁移技术说明

## 一、背景与问题

### 1.1 当前架构分析

当前项目使用本地文件系统存储上传的文件，主要实现包括：

- **普通文件上传**：使用 `multer` 中间件，文件存储在本地 `uploads` 目录
- **大文件上传**：使用 `tus-node-server` 协议，支持分片上传和断点续传，但状态存储在本地文件系统
- **文件访问**：通过 `ServeStaticModule` 提供静态文件访问
- **文件元数据**：存储在数据库 `uploads` 表中，包含文件名、路径、大小、类型、hash等信息

### 1.2 存在的问题

**单点故障风险**：
- 本地文件系统无法水平扩展，多实例部署时无法共享文件
- 服务器宕机会导致文件丢失和上传中断
- 磁盘空间受限，无法应对大量文件存储需求

**性能瓶颈**：
- 本地文件 I/O 在高并发场景下成为性能瓶颈
- 同步文件操作（如 `fs.renameSync`）阻塞事件循环
- 缺少并发控制和限流机制

**扩展性差**：
- 无法实现 CDN 加速
- 缺少自动备份和容灾机制
- 无法实现多地域部署

**成本问题**：
- 需要维护存储服务器硬件
- 需要手动管理磁盘空间
- 缺少自动扩缩容能力

### 1.3 解决方案

迁移到第三方对象存储服务（OSS），如阿里云 OSS、腾讯云 COS、AWS S3 等，实现：
- 无限扩展的存储空间
- 高可用性和容灾能力
- CDN 加速访问
- 成本优化（按需付费）
- 简化运维管理

**推荐方案**：
- **阿里云 OSS**：国内访问速度快，功能完善，文档齐全
- **腾讯云 COS**：价格优惠，与微信生态集成好

### 2.3 核心功能实现

#### 2.3.1 普通文件上传

**实现思路**：
1. 使用 `multer` 中间件接收文件（临时存储在内存或本地临时目录）
2. 调用 OSS SDK 上传文件到对象存储
3. 删除本地临时文件
4. 保存文件元数据到数据库
5. 返回 OSS 文件访问 URL

**关键代码示例**：

```typescript
import { Injectable } from '@nestjs/common';
import { UploadService } from './upload.service';
import * as OSS from 'ali-oss';

@Injectable()
export class OSSUploadService {
  private client: OSS;

  constructor(private configService: ConfigService) {
    // 初始化 OSS 客户端
    this.client = new OSS({
      region: this.configService.get('OSS_REGION'),
      accessKeyId: this.configService.get('OSS_ACCESS_KEY_ID'),
      accessKeySecret: this.configService.get('OSS_ACCESS_KEY_SECRET'),
      bucket: this.configService.get('OSS_BUCKET'),
    });
  }

  /**
   * 上传文件到 OSS
   * @param file 本地文件路径或 Buffer
   * @param fileName 目标文件名
   * @returns OSS 文件 URL
   */
  async uploadToOSS(file: Buffer | string, fileName: string): Promise<string> {
    try {
      // 生成唯一文件名（包含日期路径，便于管理）
      const date = new Date();
      const year = date.getFullYear();
      const month = String(date.getMonth() + 1).padStart(2, '0');
      const day = String(date.getDate()).padStart(2, '0');
      const objectName = `${year}/${month}/${day}/${fileName}`;

      // 上传文件到 OSS
      const result = await this.client.put(objectName, file);

      // 返回文件访问 URL
      return result.url;
    } catch (error) {
      throw new Error(`OSS 上传失败: ${error.message}`);
    }
  }

  /**
   * 删除 OSS 文件
   * @param objectName OSS 对象名称
   */
  async deleteFromOSS(objectName: string): Promise<void> {
    try {
      await this.client.delete(objectName);
    } catch (error) {
      throw new Error(`OSS 删除失败: ${error.message}`);
    }
  }

  /**
   * 获取文件签名 URL（用于私有文件访问）
   * @param objectName OSS 对象名称
   * @param expires 过期时间（秒）
   * @returns 签名 URL
   */
  async getSignedUrl(objectName: string, expires: number = 3600): Promise<string> {
    try {
      return this.client.signatureUrl(objectName, { expires });
    } catch (error) {
      throw new Error(`生成签名 URL 失败: ${error.message}`);
    }
  }
}
```

#### 2.3.2 大文件分片上传

**实现思路**：
1. 使用 OSS SDK 的分片上传功能
2. 前端计算文件 hash，查询是否已存在（秒传）
3. 如果不存在，初始化分片上传
4. 分片上传，支持断点续传
5. 合并分片，完成上传
6. 保存文件元数据到数据库

**关键代码示例**：

```typescript
import { Injectable } from '@nestjs/common';
import * as OSS from 'ali-oss';

@Injectable()
export class OSSMultipartUploadService {
  private client: OSS;

  constructor(private configService: ConfigService) {
    this.client = new OSS({
      region: this.configService.get('OSS_REGION'),
      accessKeyId: this.configService.get('OSS_ACCESS_KEY_ID'),
      accessKeySecret: this.configService.get('OSS_ACCESS_KEY_SECRET'),
      bucket: this.configService.get('OSS_BUCKET'),
    });
  }

  /**
   * 初始化分片上传
   * @param fileName 文件名
   * @param fileSize 文件大小
   * @returns 上传 ID 和分片大小
   */
  async initMultipartUpload(
    fileName: string,
    fileSize: number
  ): Promise<{ uploadId: string; partSize: number }> {
    try {
      // 生成唯一文件名
      const objectName = this.generateObjectName(fileName);

      // 计算分片大小（建议 1MB - 5MB）
      const partSize = this.calculatePartSize(fileSize);

      // 初始化分片上传
      const result = await this.client.initMultipartUpload(objectName);

      return {
        uploadId: result.uploadId,
        partSize,
      };
    } catch (error) {
      throw new Error(`初始化分片上传失败: ${error.message}`);
    }
  }

  /**
   * 上传分片
   * @param objectName 对象名称
   * @param uploadId 上传 ID
   * @param partNumber 分片编号
   * @param data 分片数据
   * @returns 分片 ETag
   */
  async uploadPart(
    objectName: string,
    uploadId: string,
    partNumber: number,
    data: Buffer
  ): Promise<string> {
    try {
      const result = await this.client.uploadPart(
        objectName,
        uploadId,
        partNumber,
        data
      );
      return result.etag;
    } catch (error) {
      throw new Error(`上传分片失败: ${error.message}`);
    }
  }

  /**
   * 完成分片上传
   * @param objectName 对象名称
   * @param uploadId 上传 ID
   * @param parts 分片列表
   * @returns 文件 URL
   */
  async completeMultipartUpload(
    objectName: string,
    uploadId: string,
    parts: Array<{ number: number; etag: string }>
  ): Promise<string> {
    try {
      const result = await this.client.completeMultipartUpload(
        objectName,
        uploadId,
        parts
      );
      return result.url;
    } catch (error) {
      throw new Error(`完成分片上传失败: ${error.message}`);
    }
  }

  /**
   * 取消分片上传
   * @param objectName 对象名称
   * @param uploadId 上传 ID
   */
  async abortMultipartUpload(objectName: string, uploadId: string): Promise<void> {
    try {
      await this.client.abortMultipartUpload(objectName, uploadId);
    } catch (error) {
      throw new Error(`取消分片上传失败: ${error.message}`);
    }
  }

  /**
   * 列出已上传的分片
   * @param objectName 对象名称
   * @param uploadId 上传 ID
   * @returns 分片列表
   */
  async listParts(
    objectName: string,
    uploadId: string
  ): Promise<Array<{ number: number; etag: string; size: number }>> {
    try {
      const result = await this.client.listParts(objectName, uploadId);
      return result.map((part) => ({
        number: part.PartNumber,
        etag: part.ETag,
        size: part.Size,
      }));
    } catch (error) {
      throw new Error(`列出分片失败: ${error.message}`);
    }
  }

  /**
   * 生成对象名称（包含日期路径）
   * @param fileName 原始文件名
   * @returns 对象名称
   */
  private generateObjectName(fileName: string): string {
    const date = new Date();
    const year = date.getFullYear();
    const month = String(date.getMonth() + 1).padStart(2, '0');
    const day = String(date.getDate()).padStart(2, '0');
    const uuid = this.generateUUID();
    const extension = fileName.split('.').pop();
    return `${year}/${month}/${day}/${uuid}.${extension}`;
  }

  /**
   * 计算分片大小
   * @param fileSize 文件大小
   * @returns 分片大小（字节）
   */
  private calculatePartSize(fileSize: number): number {
    // 根据文件大小动态计算分片大小
    // 小文件：1MB，中等文件：5MB，大文件：10MB
    if (fileSize < 100 * 1024 * 1024) {
      return 1 * 1024 * 1024; // 1MB
    } else if (fileSize < 1024 * 1024 * 1024) {
      return 5 * 1024 * 1024; // 5MB
    } else {
      return 10 * 1024 * 1024; // 10MB
    }
  }

  /**
   * 生成 UUID
   * @returns UUID 字符串
   */
  private generateUUID(): string {
    return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, (c) => {
      const r = (Math.random() * 16) | 0;
      const v = c === 'x' ? r : (r & 0x3) | 0x8;
      return v.toString(16);
    });
  }
}
```

#### 2.3.3 文件秒传

**实现思路**：
1. 前端计算文件 hash（使用 SparkMD5 或 crypto）
2. 后端查询数据库，判断文件是否已存在
3. 如果存在，直接返回已有文件的 URL
4. 如果不存在，返回需要上传

**关键代码示例**：

```typescript
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Upload } from './entities/upload.entity';

@Injectable()
export class FileHashService {
  constructor(
    @InjectRepository(Upload)
    private uploadRepository: Repository<Upload>
  ) {}

  /**
   * 根据文件 hash 和大小查询文件
   * @param hash 文件 hash
   * @param size 文件大小
   * @returns 文件信息或 null
   */
  async getFileByHash(hash: string, size: number): Promise<Upload | null> {
    try {
      const file = await this.uploadRepository.findOne({
        where: {
          hash,
          size,
        },
      });
      return file;
    } catch (error) {
      throw new Error(`查询文件失败: ${error.message}`);
    }
  }

  /**
   * 保存文件元数据
   * @param fileData 文件数据
   * @returns 保存的文件信息
   */
  async saveFileMetadata(fileData: Partial<Upload>): Promise<Upload> {
    try {
      const file = this.uploadRepository.create(fileData);
      return await this.uploadRepository.save(file);
    } catch (error) {
      throw new Error(`保存文件元数据失败: ${error.message}`);
    }
  }
}
```

## 三、OSS存储后的潜在瓶颈分析

### 3.1 网络带宽瓶颈

**问题描述**：
- 文件上传需要经过应用服务器到 OSS 的网络传输
- 在高并发场景下，应用服务器带宽可能成为瓶颈
- 跨地域上传会增加延迟

**影响**：
- 上传速度受限
- 并发上传能力受限
- 用户体验下降

**解决方案**：

1. **使用客户端直传**：
   - 前端直接上传到 OSS，不经过应用服务器
   - 应用服务器只负责生成签名 URL
   - 减轻应用服务器带宽压力

   ```typescript
   /**
    * 生成客户端直传签名 URL
    * @param fileName 文件名
    * @param expires 过期时间（秒）
    * @returns 签名 URL
    */
   async generateDirectUploadUrl(fileName: string, expires: number = 3600): Promise<{
     url: string;
     method: 'PUT' | 'POST';
     headers: Record<string, string>;
   }> {
     try {
       const objectName = this.generateObjectName(fileName);
       const signedUrl = this.client.signatureUrl(objectName, {
         method: 'PUT',
         expires,
       });

       return {
         url: signedUrl,
         method: 'PUT',
         headers: {
           'Content-Type': 'application/octet-stream',
         },
       };
     } catch (error) {
       throw new Error(`生成直传签名 URL 失败: ${error.message}`);
     }
   }
   ```

2. **使用 CDN 加速**：
   - 配置 OSS 的 CDN 加速域名
   - 用户就近访问 CDN 节点
   - 提升上传和下载速度

3. **多地域部署**：
   - 在不同地域部署应用服务器
   - 使用就近的 OSS 节点
   - 减少跨地域传输延迟

### 3.2 OSS API 限流瓶颈

**问题描述**：
- OSS 服务有 API 调用频率限制
- 高并发场景下可能触发限流
- 影响上传成功率

**影响**：
- 上传请求被拒绝
- 用户体验下降
- 业务流程中断

**解决方案**：

1. **实现请求队列**：
   - 使用消息队列（如 RabbitMQ、Kafka）管理上传请求
   - 控制并发上传数量
   - 实现重试机制

   ```typescript
   import { Injectable } from '@nestjs/common';
   import { Queue } from 'bull';
   import { InjectQueue } from '@nestjs/bull';

   @Injectable()
   export class UploadQueueService {
     constructor(
       @InjectQueue('upload') private uploadQueue: Queue
     ) {}

     /**
      * 添加上传任务到队列
      * @param uploadData 上传数据
      * @returns 任务 ID
      */
     async addUploadTask(uploadData: any): Promise<string> {
       const job = await this.uploadQueue.add('upload', uploadData, {
         attempts: 3, // 重试次数
         backoff: {
           type: 'exponential',
           delay: 2000,
         },
       });
       return job.id.toString();
     }

     /**
      * 获取任务状态
      * @param jobId 任务 ID
      * @returns 任务状态
      */
     async getJobStatus(jobId: string): Promise<any> {
       const job = await this.uploadQueue.getJob(jobId);
       if (!job) {
         return null;
       }
       return {
         id: job.id,
         state: await job.getState(),
         progress: job.progress(),
         data: job.data,
       };
     }
   }
   ```

2. **使用限流中间件**：
   - 使用 `@nestjs/throttler` 实现请求限流
   - 控制单个用户的上传频率
   - 防止恶意上传

   ```typescript
   import { ThrottlerModule } from '@nestjs/throttler';

   @Module({
     imports: [
       ThrottlerModule.forRoot([{
         ttl: 60000, // 60 秒
         limit: 10, // 最多 10 次请求
       }]),
     ],
   })
   export class AppModule {}
   ```

3. **优化分片上传策略**：
   - 合理设置分片大小
   - 减少分片数量
   - 降低 API 调用次数

### 3.3 成本优化瓶颈

**问题描述**：
- OSS 存储费用随着文件数量和大小增长
- 流量费用可能很高
- 请求次数费用不容忽视

**影响**：
- 运营成本增加
- 预算超支风险
- 需要精细化管理

**解决方案**：

1. **生命周期管理**：
   - 设置文件过期时间
   - 自动删除过期文件
   - 降低存储成本

   ```typescript
   /**
    * 设置文件生命周期规则
    * @param days 过期天数
    */
   async setLifecycleRule(days: number): Promise<void> {
     try {
       await this.client.putBucketLifecycle(this.configService.get('OSS_BUCKET'), {
         rules: [
           {
             id: 'expire-files',
             status: 'Enabled',
             expiration: {
               days,
             },
           },
         ],
       });
     } catch (error) {
   ```

### 3.4 数据一致性瓶颈

**问题描述**：
- 文件上传到 OSS 后，数据库元数据可能更新失败
- 出现文件存在但数据库无记录的情况
- 影响文件管理和查询

**影响**：
- 文件管理混乱
- 查询结果不准确
- 数据不一致

**解决方案**：

1. **使用事务**：
   - 将文件上传和数据库更新放在同一个事务中
   - 确保数据一致性
   - 失败时回滚

   ```typescript
   import { Injectable } from '@nestjs/common';
   import { DataSource } from 'typeorm';

   @Injectable()
   export class TransactionUploadService {
     constructor(private dataSource: DataSource) {}

     /**
      * 事务上传文件
      * @param fileData 文件数据
      * @returns 上传结果
      */
     async uploadWithTransaction(fileData: any): Promise<any> {
       const queryRunner = this.dataSource.createQueryRunner();
       await queryRunner.connect();
       await queryRunner.startTransaction();

       try {
         // 1. 上传文件到 OSS
         const url = await this.uploadToOSS(fileData.buffer, fileData.fileName);

         // 2. 保存元数据到数据库
         const upload = queryRunner.manager.create(Upload, {
           fileName: fileData.fileName,
           url,
           size: fileData.size,
           mimetype: fileData.mimetype,
           hash: fileData.hash,
         });

         const savedUpload = await queryRunner.manager.save(upload);

         // 3. 提交事务
         await queryRunner.commitTransaction();

         return savedUpload;
       } catch (error) {
         // 4. 回滚事务
         await queryRunner.rollbackTransaction();

         // 5. 删除已上传的文件（如果上传成功但数据库失败）
         if (url) {
           try {
             await this.deleteFromOSS(this.extractObjectName(url));
           } catch (deleteError) {
             console.error('删除 OSS 文件失败:', deleteError);
           }
         }

         throw error;
       } finally {
         // 6. 释放连接
         await queryRunner.release();
       }
     }
   }
   ```

2. **文件类型验证**：
   - 严格验证文件类型
   - 检查文件内容（魔数）

   ```typescript
   import { createReadStream } from 'fs';
   import * as fileType from 'file-type';

   @Injectable()
   export class FileValidationService {
     /**
      * 验证文件类型
      * @param filePath 文件路径
      * @param expectedType 期望的文件类型
      * @returns 是否验证通过
      */
     async validateFileType(
       filePath: string,
       expectedType: string
     ): Promise<boolean> {
       try {
         const stream = createReadStream(filePath);
         const type = await fileType.fromStream(stream);

         if (!type) {
           return false;
         }

         return type.mime === expectedType;
       } catch (error) {
         throw new Error(`验证文件类型失败: ${error.message}`);
       }
     }

     /**
      * 获取文件魔数
      * @param buffer 文件 Buffer
      * @returns 文件类型
      */
     getFileMagicNumber(buffer: Buffer): string | null {
       const magicNumbers = {
         'image/jpeg': [0xFF, 0xD8, 0xFF],
         'image/png': [0x89, 0x50, 0x4E, 0x47],
         'image/gif': [0x47, 0x49, 0x46],
         'application/pdf': [0x25, 0x50, 0x44, 0x46],
       };

       for (const [mime, magic] of Object.entries(magicNumbers)) {
         if (buffer.slice(0, magic.length).equals(Buffer.from(magic))) {
           return mime;
         }
       }

       return null;
     }
   }
   ```

### 5.4 常见陷阱

1. **陷阱一：忽略文件类型验证**
   - 只检查文件扩展名，不检查文件内容
   - 可能导致恶意文件上传

2. **陷阱二：不处理并发冲突**
   - 多个用户同时上传同名文件
   - 可能导致文件覆盖
   - 解决方案：使用 UUID 生成唯一文件名

3. **陷阱三：忽略大文件上传超时**
   - 大文件上传时间过长
   - 可能导致请求超时
   - 解决方案：使用分片上传，增加超时时间

4. **陷阱四：不清理未完成的上传**
   - 分片上传中断后未清理
   - 导致存储空间浪费
   - 解决方案：实现定时清理任务

5. **陷阱五：忽略 CDN 缓存失效**
   - 文件更新后 CDN 缓存未失效
   - 用户访问到旧文件
   - 解决方案：使用版本号或文件 hash 作为文件名
