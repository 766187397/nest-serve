# 数据库连接池优化技术说明

## 一、背景与问题

### 1.1 为什么需要连接池优化

在高并发场景下，数据库连接是宝贵的资源。每次数据库操作都创建新连接会带来以下问题：

- **性能瓶颈**：建立数据库连接是昂贵的操作，需要TCP三次握手、身份验证等，耗时较长
- **资源浪费**：频繁创建和销毁连接消耗大量CPU和内存资源
- **连接泄漏**：未正确关闭的连接会导致数据库连接数耗尽
- **并发限制**：数据库服务器对最大连接数有限制，超过限制会导致请求失败

### 1.2 连接池的作用

连接池通过预先创建并维护一组数据库连接，解决了上述问题：

- **性能提升**：复用现有连接，避免频繁创建和销毁的开销
- **资源优化**：控制连接数量，避免资源耗尽
- **并发控制**：限制最大连接数，保护数据库服务器
- **负载均衡**：合理分配连接资源，提高系统吞吐量

### 1.3 本方案解决的痛点

- **缺少连接池配置**：使用数据库驱动默认配置，无法满足高并发需求
- **参数不合理**：连接池大小、超时时间等参数未根据实际负载调整
- **缺少监控**：无法实时了解连接池状态，难以发现和解决问题
- **无读写分离**：所有请求都打到主库，无法充分利用从库资源

```typescript
  DB_MAX_LIFETIME?: number;

  DB_SLAVE_HOST?: string;
  // 从库端口
  DB_SLAVE_PORT?: number;

/** PostgreSQL配置类型 */
export interface PgConfig {
  DB_TYPE: string;
  DB_HOST: string;
  DB_PORT: number;
  DB_USER: string;
  DB_PASSWORD: string;
  DB_DATABASE: string;
  DB_POOL_SIZE?: number;
  DB_POOL_MIN?: number;
  DB_CONNECTION_TIMEOUT?: number;
  DB_IDLE_TIMEOUT?: number;
  DB_MAX_LIFETIME?: number;
  DB_SLAVE_HOST?: string;
  DB_SLAVE_PORT?: number;
}

/** mysql 配置 */
const mysqlConfig: MysqlConfig = {
  DB_TYPE: 'mysql',
  DB_HOST: env.DB_HOST || '127.0.0.1',
  DB_PORT: Number(env.DB_PORT) || 3306,
  DB_USER: env.DB_USER || 'root',
  DB_PASSWORD: env.DB_PASSWORD || '123456',
  DB_DATABASE: env.DB_DATABASE || 'nest-serve',
  // 连接池大小: 建议值为 (CPU核心数 * 2) + 有效磁盘数
  // 如果环境变量未配置，则使用默认值计算
  DB_POOL_SIZE: Number(env.DB_POOL_SIZE) || (cpuCores * 2 + 1),
  // 最小连接数: 保持连接池中至少有这些数量的连接
  // 默认为连接池大小的1/4，至少为2
  DB_POOL_MIN: Number(env.DB_POOL_MIN) || Math.max(2, Math.floor((cpuCores * 2 + 1) / 4)),
  // 连接超时: 等待获取连接的最大时间(毫秒)
  // 默认30秒，避免长时间等待导致请求超时
  DB_CONNECTION_TIMEOUT: Number(env.DB_CONNECTION_TIMEOUT) || 30000,
  // 空闲超时: 连接在池中空闲超过此时间将被释放(毫秒)
  // 默认5分钟，及时释放不活跃连接
  DB_IDLE_TIMEOUT: Number(env.DB_IDLE_TIMEOUT) || 300000,
  // 连接最大生命周期: 连接在池中的最大存活时间(毫秒)
  // 默认30分钟，定期刷新连接避免长时间使用同一连接
  DB_MAX_LIFETIME: Number(env.DB_MAX_LIFETIME) || 1800000,
  // 从库地址: 如果未配置则使用主库地址
  DB_SLAVE_HOST: env.DB_SLAVE_HOST || env.DB_HOST || '127.0.0.1',
  // 从库端口: 如果未配置则使用主库端口
  DB_SLAVE_PORT: Number(env.DB_SLAVE_PORT) || Number(env.DB_PORT) || 3306,
};
```

**关键点解析**：

1. **CPU核心数计算**：使用 `os.cpus().length` 获取CPU核心数，这是计算连接池大小的基础
2. **连接池大小公式**：`(CPU核心数 * 2) + 有效磁盘数`，这是业界推荐的公式
3. **最小连接数**：保持连接池中至少有这些数量的连接，避免频繁创建连接
4. **超时配置**：连接超时、空闲超时、最大生命周期三个参数协同工作，确保连接健康

### 2.3 连接池监控服务

#### 文件位置：`src/common/service/connection-pool-monitor.service.ts`

```typescript
import { Injectable, Logger } from '@nestjs/common';
import { InjectDataSource } from '@nestjs/typeorm';
import { DataSource } from 'typeorm';

interface PoolStats {
  totalConnections: number;      // 总连接数
  idleConnections: number;       // 空闲连接数
  activeConnections: number;     // 活跃连接数
  waitingRequests: number;      // 等待请求数
  maxConnections: number;        // 最大连接数
  minConnections: number;        // 最小连接数
}

@Injectable()
export class ConnectionPoolMonitorService {
  private readonly logger = new Logger(ConnectionPoolMonitorService.name);
  // 存储历史数据，每个连接池最多保留100条记录
  private readonly statsHistory: Map<string, Array<{ timestamp: number; stats: PoolStats }>> = new Map();
  private readonly MAX_HISTORY_SIZE = 100;

  constructor(
    @InjectDataSource() private readonly mainDataSource: DataSource,
    @InjectDataSource('logger') private readonly loggerDataSource: DataSource,
  ) {}

  async getMainPoolStats(): Promise<PoolStats> {
    return this.getPoolStats(this.mainDataSource, 'main');
  }

  async getLoggerPoolStats(): Promise<PoolStats> {
    return this.getPoolStats(this.loggerDataSource, 'logger');
  }

  async getAllPoolStats(): Promise<{ main: PoolStats; logger: PoolStats }> {
    const [main, logger] = await Promise.all([
      this.getMainPoolStats(),
      this.getLoggerPoolStats()
    ]);
    return { main, logger };
  }

  private async getPoolStats(dataSource: DataSource, poolName: string): Promise<PoolStats> {
    try {
      // 获取数据库驱动
      const driver = dataSource.driver;
      // 获取连接池对象，不同驱动存储位置不同
      const pool: any = driver.master || driver.pool;

      if (!pool) {
        this.logger.warn(`Connection pool not found for ${poolName}`);
        return this.getDefaultStats();
      }

      let stats: PoolStats;

      // MySQL连接池统计
      if (pool._allConnections) {
        stats = this.getMysqlPoolStats(pool);
      }
      // PostgreSQL连接池统计
      else if (pool.totalCount !== undefined) {
        stats = this.getPostgresPoolStats(pool);
      }
      // 其他情况返回默认值
      else {
        stats = this.getDefaultStats();
      }

      // 记录历史数据
      this.recordStats(poolName, stats);
      return stats;
    } catch (error) {
      this.logger.error(`Error getting pool stats for ${poolName}:`, error);
      return this.getDefaultStats();
    }
  }

  private getMysqlPoolStats(pool: any): PoolStats {
    return {
      // 总连接数: 所有连接的数量
      totalConnections: pool._allConnections?.length || 0,
      // 空闲连接数: 可用的连接数量
      idleConnections: pool._freeConnections?.length || 0,
      // 活跃连接数: 正在使用的连接数量
      activeConnections: (pool._allConnections?.length || 0) - (pool._freeConnections?.length || 0),
      // 等待请求数: 等待获取连接的请求数量
      waitingRequests: pool._connectionQueue?.length || 0,
      // 最大连接数: 连接池的最大连接数
      maxConnections: pool._connectionLimit || 10,
      // 最小连接数: MySQL连接池不维护最小连接数
      minConnections: 0,
    };
  }

  private getPostgresPoolStats(pool: any): PoolStats {
    return {
      // 总连接数: 连接池中的总连接数
      totalConnections: pool.totalCount || 0,
      // 空闲连接数: 空闲的连接数
      idleConnections: pool.idleCount || 0,
      // 活跃连接数: 正在使用的连接数
      activeConnections: (pool.totalCount || 0) - (pool.idleCount || 0),
      // 等待请求数: 等待获取连接的请求数
      waitingRequests: pool.waitingCount || 0,
      // 最大连接数: 连接池的最大连接数
      maxConnections: pool.options?.max || 10,
      // 最小连接数: 连接池的最小连接数
      minConnections: pool.options?.min || 2,
    };
  }

  private getDefaultStats(): PoolStats {
    return {
      totalConnections: 0,
      idleConnections: 0,
      activeConnections: 0,
      waitingRequests: 0,
      maxConnections: 10,
      minConnections: 2,
    };
  }

  private recordStats(poolName: string, stats: PoolStats): void {
    // 如果该连接池没有历史记录，初始化空数组
    if (!this.statsHistory.has(poolName)) {
      this.statsHistory.set(poolName, []);
    }

    const history = this.statsHistory.get(poolName);
    // 添加新的记录
    history.push({ timestamp: Date.now(), stats });

    // 如果历史记录超过最大数量，删除最旧的记录
    if (history.length > this.MAX_HISTORY_SIZE) {
      history.shift();
    }
  }

  getStatsHistory(poolName: string, limit: number = 10): Array<{ timestamp: number; stats: PoolStats }> {
    const history = this.statsHistory.get(poolName) || [];
    // 返回最近的N条记录
    return history.slice(-limit);
  }

  async getPoolHealthStatus(): Promise<{
    main: { healthy: boolean; issues: string[] };
    logger: { healthy: boolean; issues: string[] };
  }> {
    const [mainStats, loggerStats] = await Promise.all([
      this.getMainPoolStats(),
      this.getLoggerPoolStats()
    ]);

    return {
      main: this.checkHealth(mainStats, 'main'),
      logger: this.checkHealth(loggerStats, 'logger'),
    };
  }

  private checkHealth(stats: PoolStats, poolName: string): { healthy: boolean; issues: string[] } {
    const issues: string[] = [];

    // 检查连接池使用率是否过高
    if (stats.activeConnections / stats.maxConnections > 0.9) {
      issues.push(`${poolName}: 连接池使用率超过90% (${stats.activeConnections}/${stats.maxConnections})`);
    }

    // 检查等待请求数是否过多
    if (stats.waitingRequests > 10) {
      issues.push(`${poolName}: 等待连接的请求数过多 (${stats.waitingRequests})`);
    }

    // 检查空闲连接是否过多
    if (stats.idleConnections > stats.maxConnections * 0.5) {
      issues.push(`${poolName}: 空闲连接过多 (${stats.idleConnections}/${stats.maxConnections})`);
    }

    return {
      healthy: issues.length === 0,
      issues,
    };
  }

  async logPoolStats(): Promise<void> {
    const [main, logger] = await Promise.all([
      this.getMainPoolStats(),
      this.getLoggerPoolStats()
    ]);

    this.logger.log('Main DB Pool Stats:', {
      total: main.totalConnections,
      active: main.activeConnections,
      idle: main.idleConnections,
      waiting: main.waitingRequests,
      max: main.maxConnections,
    });

    this.logger.log('Logger DB Pool Stats:', {
      total: logger.totalConnections,
      active: logger.activeConnections,
      idle: logger.idleConnections,
      waiting: logger.waitingRequests,
      max: logger.maxConnections,
    });
  }
}
```

**关键点解析**：

1. **多数据库支持**：同时支持MySQL和PostgreSQL连接池监控
2. **历史数据记录**：使用Map存储历史数据，最多保留100条
3. **健康检查**：检查连接池使用率、等待请求数、空闲连接数
4. **异常处理**：获取连接池状态失败时返回默认值，避免影响业务

```typescript
import { Controller, Get } from '@nestjs/common';
import { ApiTags, ApiOperation, ApiResponse } from '@nestjs/swagger';
import { ConnectionPoolMonitorService } from '../service/connection-pool-monitor.service';

@ApiTags('连接池监控')
@Controller('health/pool')
export class ConnectionPoolMonitorController {
  constructor(
    private readonly poolMonitorService: ConnectionPoolMonitorService
  ) {}

  @Get('main')
  @ApiOperation({ summary: '获取主数据库连接池状态' })
  @ApiResponse({ status: 200, description: '成功获取连接池状态' })
  async getMainPoolStats() {
    return await this.poolMonitorService.getMainPoolStats();
  }

  @Get('logger')
  @ApiOperation({ summary: '获取日志数据库连接池状态' })
  @ApiResponse({ status: 200, description: '成功获取连接池状态' })
  async getLoggerPoolStats() {
    return await this.poolMonitorService.getLoggerPoolStats();
  }

  @Get('all')
  @ApiOperation({ summary: '获取所有数据库连接池状态' })
  @ApiResponse({ status: 200, description: '成功获取所有连接池状态' })
  async getAllPoolStats() {
    return await this.poolMonitorService.getAllPoolStats();
  }

  @Get('health')
  @ApiOperation({ summary: '获取连接池健康状态' })
  @ApiResponse({ status: 200, description: '成功获取健康状态' })
  async getPoolHealth() {
    return await this.poolMonitorService.getPoolHealthStatus();
  }

  @Get('history/main')
  @ApiOperation({ summary: '获取主数据库连接池历史数据' })
  @ApiResponse({ status: 200, description: '成功获取历史数据' })
  async getMainPoolHistory() {
    return this.poolMonitorService.getStatsHistory('main', 20);
  }

  @Get('history/logger')
  @ApiOperation({ summary: '获取日志数据库连接池历史数据' })
  @ApiResponse({ status: 200, description: '成功获取历史数据' })
  async getLoggerPoolHistory() {
    return this.poolMonitorService.getStatsHistory('logger', 20);
  }
}
```

**关键点解析**：

1. **RESTful API设计**：使用GET方法获取连接池状态，符合RESTful规范
2. **Swagger文档**：使用@ApiTags、@ApiOperation、@ApiResponse生成API文档
3. **模块化设计**：每个接口对应服务层的一个方法，职责清晰

### 3.5 完整示例：定时监控连接池

```typescript
import { Injectable, Logger } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';
import { ConnectionPoolMonitorService } from './connection-pool-monitor.service';

@Injectable()
export class PoolMonitorScheduler {
  private readonly logger = new Logger(PoolMonitorScheduler.name);

  constructor(
    private readonly poolMonitor: ConnectionPoolMonitorService,

      waiting: logger.waitingRequests,
      max: logger.maxConnections,
    });
  }

    }
  }
```

## 五、注意事项

### 5.1 连接池大小配置

**常见陷阱**：
- ❌ 连接池设置过大：导致数据库服务器负载过高
- ❌ 连接池设置过小：导致请求排队，响应变慢
- ❌ 所有服务使用相同配置：未考虑不同服务的负载特点

**最佳实践**：
- ✅ 根据CPU核心数计算连接池大小：`(CPU核心数 * 2) + 有效磁盘数`
- ✅ 根据实际负载调整：监控连接池使用率，动态调整
- ✅ 考虑数据库服务器限制：确保总连接数不超过数据库最大连接数

**示例**：
```typescript
// 8核CPU，1块磁盘
const poolSize = 8 * 2 + 1 = 17

// 16核CPU，2块磁盘
const poolSize = 16 * 2 + 2 = 34

// 4核CPU，1块磁盘
const poolSize = 4 * 2 + 1 = 9
```

### 5.2 超时时间配置

**常见陷阱**：
- ❌ 连接超时设置过短：导致正常请求超时
- ❌ 连接超时设置过长：导致请求堆积
- ❌ 空闲超时设置过短：频繁创建和销毁连接

**最佳实践**：
- ✅ 连接超时：30秒，平衡响应时间和资源占用
- ✅ 空闲超时：5分钟，及时释放不活跃连接
- ✅ 连接生命周期：30分钟，定期刷新连接

**示例**：
```typescript
// 快速响应的API
DB_CONNECTION_TIMEOUT=10000  // 10秒
DB_IDLE_TIMEOUT=180000       // 3分钟
DB_MAX_LIFETIME=900000      // 15分钟

// 复杂查询的报表系统
DB_CONNECTION_TIMEOUT=60000  // 60秒
DB_IDLE_TIMEOUT=600000       // 10分钟
DB_MAX_LIFETIME=3600000     // 60分钟
```

### 5.3 连接池监控

**常见陷阱**：
- ❌ 只监控连接数，不监控等待请求数
- ❌ 监控频率过高，影响性能
- ❌ 监控数据不持久化，无法分析历史趋势

**最佳实践**：
- ✅ 监控关键指标：总连接数、空闲连接数、活跃连接数、等待请求数
- ✅ 监控频率适中：每5-10分钟采集一次
- ✅ 持久化监控数据：存储到时序数据库，便于分析

**示例**：
```typescript
// 监控指标
const metrics = {
  totalConnections: pool.totalCount,
  idleConnections: pool.idleCount,
  activeConnections: pool.totalCount - pool.idleCount,
  waitingRequests: pool.waitingCount,
  maxConnections: pool.options.max,
};

// 存储到Prometheus
prometheusMetrics.gauge('db_pool_total', metrics.totalConnections);
prometheusMetrics.gauge('db_pool_idle', metrics.idleConnections);
prometheusMetrics.gauge('db_pool_active', metrics.activeConnections);
prometheusMetrics.gauge('db_pool_waiting', metrics.waitingRequests);
```

### 5.4 读写分离

**常见陷阱**：
- ❌ 所有请求都打到主库，导致主库负载过高
- ❌ 从库数据延迟，导致读取到旧数据
- ❌ 从库故障时没有降级方案

**最佳实践**：
- ✅ 读操作使用从库，写操作使用主库
- ✅ 监控从库延迟，超过阈值时切换到主库
- ✅ 实现从库故障自动切换

**示例**：
```typescript
// 读操作使用从库
async findUser(id: number) {
  return this.userRepository.find({
    where: { id },
    // 使用从库连接
    connection: 'slave',
  });
}

// 写操作使用主库
async createUser(userData: CreateUserDto) {
  return this.userRepository.save(userData);
  // 使用主库连接
}

// 监控从库延迟
async checkSlaveDelay() {
  const masterTime = await this.getMasterTime();
  const slaveTime = await this.getSlaveTime();
  const delay = masterTime - slaveTime;
  
  if (delay > 5000) {
    // 延迟超过5秒，切换到主库
    this.useMasterForReads = true;
  }
}
```

### 5.5 性能优化

**常见陷阱**：
- ❌ 频繁创建和销毁连接
- ❌ 连接泄漏导致连接池耗尽
- ❌ 查询未优化导致连接占用时间过长

**最佳实践**：
- ✅ 使用连接池复用连接
- ✅ 及时释放连接，避免连接泄漏
- ✅ 优化SQL查询，减少连接占用时间

**示例**：
```typescript
// ✅ 正确：使用连接池
async getUsers() {
  // 连接会自动从连接池获取
  const users = await this.userRepository.find();
  // 连接会自动返回到连接池
  return users;
}

// ❌ 错误：手动管理连接
async getUsers() {
  const connection = await this.dataSource.getConnection();
  try {
    const users = await connection.query('SELECT * FROM users');
    return users;
  } finally {
    // 必须手动释放连接
    await connection.release();
  }
}

// ✅ 优化查询，减少连接占用时间
async getUsers() {
  return this.userRepository.find({
    select: ['id', 'name', 'email'],  // 只查询需要的字段
    take: 100,                        // 限制返回数量
    cache: true,                       // 使用缓存
  });
}
```

### 5.6 异常处理

**常见陷阱**：
- ❌ 连接池异常导致整个服务不可用
- ❌ 异常信息不详细，难以排查问题
- ❌ 异常恢复不及时，影响业务

**最佳实践**：
- ✅ 连接池异常时降级处理
- ✅ 记录详细的异常日志
- ✅ 实现自动恢复机制

**示例**：
```typescript
async getUsers() {
  try {
    return await this.userRepository.find();
  } catch (error) {
    // 连接池异常
    if (error.code === 'ETIMEDOUT') {
      this.logger.error('数据库连接超时', error);
      // 降级：返回缓存数据
      return await this.cacheService.get('users');
    }
    // 其他异常
    throw error;
  }
}

// 自动恢复机制
@Cron(CronExpression.EVERY_MINUTE)
async checkConnection() {
  try {
    await this.dataSource.query('SELECT 1');
    this.isConnected = true;
  } catch (error) {
    this.logger.error('数据库连接失败', error);
    this.isConnected = false;
    // 尝试重新连接
    await this.reconnect();
  }
}
```
