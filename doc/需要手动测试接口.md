# 需要手动测试接口

## 测试说明
1. 首先调用登录接口获取token：`POST /api/v1/admin/users/login`
2. 在后续需要token的接口请求头中添加：`Authorization: Bearer {token}`
3. 本文档只记录在接口逻辑中真正使用了`req.userInfo`（即token转换为用户信息）的接口

## 测试账号信息
### 默认测试账号
- **管理员账号**:
  - 账号: `admin`
  - 密码: `123456`
  - 平台: `admin`

## 1. 邮箱管理模块

### 1.1 发送邮箱
- **接口路径**: `POST /api/v1/admin/email/send/email`
- **请求头**: 
  - `x-platform`: 平台标识（如：admin、web）
  - `Authorization`: Bearer {token}
- **请求体**: SendEmail
  ```json
  {
    "email": "test@example.com",
    "templateId": "模板ID",
    "variables": {
      "code": "123456"
    }
  }
  ```
- **说明**: 发送邮箱，接口逻辑中使用`req.userInfo`获取当前用户信息
- **token用途**: 获取当前用户信息用于发送邮件

## 2. 路由管理模块

### 2.1 根据登录用户的角色ids获取路由
- **接口路径**: `GET /api/v1/admin/routes/by-role`
- **请求头**: 
  - `x-platform`: 平台标识（如：admin、web）
  - `Authorization`: Bearer {token}
- **查询参数**: 
  - `type`: 路由类型（可选，如：menu）
- **说明**: 根据登录用户的角色ids获取路由，接口逻辑中使用`req.userInfo.roles`获取用户角色信息
- **token用途**: 获取当前用户的角色信息，用于过滤路由权限

## 3. 公告管理模块

### 3.1 查询公告列表(分页,查询当前用户和角色权限对应的公告)
- **接口路径**: `GET /api/v1/admin/notice/page/userOrRole`
- **请求头**: 
  - `x-platform`: 平台标识（如：admin、web）
  - `Authorization`: Bearer {token}
- **查询参数**: FindNoticeDtoByPageByUserOrRole
  ```json
  {
    "page": 1,
    "pageSize": 10,
    "title": "公告标题（可选）",
    "status": "状态（可选）"
  }
  ```
- **说明**: 查询公告列表，接口逻辑中使用`req.userInfo.id`和`req.userInfo.roles`获取用户信息
- **token用途**: 获取当前用户的ID和角色信息，用于过滤公告权限

### 3.2 标记公告为已读
- **接口路径**: `POST /api/v1/admin/notice/read/:id`
- **请求头**: 
  - `Authorization`: Bearer {token}
- **路径参数**: 
  - `id`: 公告ID
- **说明**: 标记公告为已读，接口逻辑中使用`req.userInfo.id`获取用户ID
- **token用途**: 获取当前用户的ID，用于标记已读状态

## 注意事项
1. 所有接口都需要先登录获取token
2. token有效期内可重复使用
3. token过期后需要调用刷新接口：`GET /api/v1/admin/users/refresh-token`
4. 测试时请确保数据库中有对应的测试数据
