# 国际化

国际化是指设计和开发能够适应不同语言和文化习惯的应用程序的过程。在本教程中，我们将学习如何在项目中实现国际化功能，使应用能够支持多种语言。

## 1. 国际化概述

### 1.1 为什么需要国际化

- **扩大用户群体**：支持多种语言可以吸引更多来自不同地区的用户
- **提升用户体验**：用户可以使用自己熟悉的语言使用应用
- **符合本地化法规**：某些地区要求应用提供本地语言版本
- **增强品牌形象**：支持多语言显示了企业的全球化视野

### 1.2 国际化与本地化的区别

- **国际化（i18n）**：设计和开发能够支持多种语言的应用架构
- **本地化（l10n）**：将应用中的文本和资源翻译成特定语言和地区的版本

## 2. 技术选型

在本项目中，我们将使用以下技术实现国际化：

- **nestjs-i18n**：NestJS 官方推荐的国际化库，提供了强大的翻译功能
- **JSON**：用于存储翻译文本，格式简单易维护

### 3.3 配置 i18n 模块

在 `app.module.ts` 中配置 i18n 模块：

```typescript
import { Module } from '@nestjs/common';
import { I18nModule } from 'nestjs-i18n';
import * as path from 'path';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [
    I18nModule.forRoot({
      fallbackLanguage: 'zh', // 默认语言
      loaderOptions: {
        path: path.join(__dirname, '/i18n/'),
        watch: true, // 监听文件变化，自动重新加载
      },
      parserOptions: {
        useAcceptLanguageHeader: true, // 从请求头 Accept-Language 检测语言
        useQueryString: true, // 从查询参数检测语言
        queryStringParamName: 'lang', // 查询参数名称，默认为 'lang'
      },
    }),
    // 其他模块
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

## 4. 实现国际化功能

### 4.1 在控制器中使用

```typescript
import { Controller, Get, Post, Body, Param, Delete, I18n, I18nContext } from '@nestjs/common';
import { UsersService } from './users.service';
import { CreateUserDto } from './dto/create-user.dto';
import { UpdateUserDto } from './dto/update-user.dto';

@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Post()
  async create(@Body() createUserDto: CreateUserDto, @I18n() i18n: I18nContext) {
    try {
      const user = await this.usersService.create(createUserDto);
      return {
        success: true,
        message: i18n.t('user.create.success'),
        data: user,
      };
    } catch (error) {
      return {
        success: false,
        message: i18n.t('user.create.error'),
        error: error.message,
      };
    }
  }

  @Get(':id')
  async findOne(@Param('id') id: string, @I18n() i18n: I18nContext) {
    const user = await this.usersService.findOne(id);
    if (!user) {
      return {
        success: false,
        message: i18n.t('user.not_found'),
      };
    }
    return {
      success: true,
      data: user,
    };
  }

  // 其他方法...
}
```

### 4.3 在异常过滤器中使用

```typescript
import { ExceptionFilter, Catch, ArgumentsHost, HttpException } from '@nestjs/common';
import { Response } from 'express';
import { I18nService } from 'nestjs-i18n';

@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  constructor(private readonly i18n: I18nService) {}

  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();
    const status = exception.getStatus();
    
    // 获取语言
    const lang = request.headers['accept-language']?.split(',')[0] || 'zh';
    
    // 根据状态码返回对应的错误信息
    let message: string;
    switch (status) {
      case 404:
        message = this.i18n.translate('common.not_found', { lang });
        break;
      case 401:
        message = this.i18n.translate('common.unauthorized', { lang });
        break;
      case 403:
        message = this.i18n.translate('common.forbidden', { lang });
        break;
      default:
        message = exception.message || this.i18n.translate('common.error', { lang });
    }

    response
      .status(status)
      .json({
        success: false,
        message,
        statusCode: status,
        timestamp: new Date().toISOString(),
        path: request.url,
      });
  }
}
```

## 6. 前端集成

### 6.1 前端配置

在前端应用中，我们可以通过以下方式实现国际化：

1. **保存用户语言偏好**：将用户选择的语言保存到本地存储或Cookie中
2. **请求头设置**：在发送API请求时，将用户语言作为 `Accept-Language` 头传递给后端
3. **前端翻译**：对于静态文本，可以在前端使用i18n库进行翻译

### 6.2 示例：前端请求头设置

**使用axios**：

```javascript
import axios from 'axios';

// 从本地存储获取语言
const lang = localStorage.getItem('language') || 'zh';

// 创建axios实例
const api = axios.create({
  baseURL: 'http://localhost:3000',
  headers: {
    'Accept-Language': lang,
  },
});

// 切换语言时更新请求头
function changeLanguage(lang) {
  localStorage.setItem('language', lang);
  api.defaults.headers['Accept-Language'] = lang;
}
```

## 7. 最佳实践

### 7.1 翻译文件组织

- **按模块组织**：将不同模块的翻译文本分开存储，如 `user.json`、`auth.json` 等
- **使用嵌套结构**：使用嵌套结构组织翻译键，提高可读性和维护性
- **保持一致性**：使用统一的命名约定，如 `module.action.result`

### 7.2 翻译键命名

- **使用语义化键名**：如 `user.create.success` 而不是 `msg1`
- **避免硬编码**：所有用户可见的文本都应该通过翻译键获取
- **使用一致的命名风格**：如驼峰命名或蛇形命名

### 7.3 性能优化

- **启用缓存**：对于大型应用，可以启用翻译缓存
- **按需加载**：只加载当前语言的翻译文件
- **避免在循环中使用翻译函数**：尽量在循环外获取翻译文本

### 7.4 测试

- **测试所有语言**：确保所有语言版本都能正常工作
- **测试边缘情况**：如缺失翻译键、无效语言等
- **测试语言切换**：确保语言切换功能正常工作

## 8. 扩展功能

### 8.1 支持更多语言

要支持更多语言，只需添加相应的翻译文件即可：

```bash
mkdir -p i18n/ja i18n/ko
```

然后创建 `i18n/ja/translation.json` 和 `i18n/ko/translation.json` 文件。

### 8.2 复数形式支持

`nestjs-i18n` 支持复数形式，如：

```json
{
  "apples": "You have {count, plural, =0 {no apples} =1 {one apple} other {{count} apples}}"
}
```

### 8.3 日期和时间格式化

可以使用 `Intl` API 格式化日期和时间：

```typescript
const date = new Date();
const formatter = new Intl.DateTimeFormat('zh-CN', {
  year: 'numeric',
  month: 'long',
  day: 'numeric',
  hour: '2-digit',
  minute: '2-digit',
  second: '2-digit',
});
const formattedDate = formatter.format(date);
```
